# Glasswell Field Test Checklist

Use this checklist during the first employee field test. Run it once in Demo Mode before using Cloud Mode.

## Pre-Shift

- Install the PWA on an iPhone-size device.
- Open the app outdoors with screen brightness high.
- Confirm Demo Mode or Cloud Mode is clearly visible.
- Confirm the employee can see their assigned territory.
- Confirm map, territory polygons, streets, lead pins, and filters load.

## Field Workflow

- Start shift.
- Start route.
- Confirm the route list appears.
- Confirm a next street recommendation appears.
- Start street from the route.
- Knock houses with the large `+` house counter.
- Record at least one quick result.
- Create a lead.
- Create a follow-up.
- Book a job.
- Complete street.
- Confirm the route refreshes after street completion.
- End shift.
- Review the shift summary.

## Resilience

- Refresh the page during an active shift.
- Confirm the active shift resumes.
- Confirm the active street resumes if one was active.
- Use the app with weak/no internet.
- Confirm Demo Mode still opens.
- Confirm saved local work remains after refresh.

## Data Safety

- Export JSON backup.
- Reset demo data after reading the confirmation.
- Restore backup after reading the confirmation.
- Confirm restored leads, streets, and shift state return.

## Manager Safety

- Try overriding a street lock.
- Confirm the override confirmation appears.
- Try marking a street do not knock.
- Confirm the do-not-knock confirmation appears.
- Confirm employees cannot access owner-only dashboard routes.

## Notes To Record

- Any buttons that are too small in sunlight.
- Any bottom sheet that covers the control being used.
- Any confusing wording in route recommendations.
- Any missing toast or unclear error state.
- Any workflow that takes too many taps.
